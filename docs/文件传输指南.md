## 📊 传输协议

### 上传流程

```
客户端                          服务端
  |                               |
  |--- file_transfer_start ------>|
  |     (文件信息、哈希、大小)     |
  |                               |
  |--- file_transfer_chunk ------>|
  |     (分块 0, Base64 编码)        |
  |                               |
  |--- file_transfer_chunk ------>|
  |     (分块 1, Base64 编码)        |
  |                               |
  |          ...                  |
  |                               |
  |--- file_transfer_chunk ------>|
  |     (分块 N, Base64 编码)        |
  |                               |
  |--- file_transfer_complete --->|
  |     (完成信号、最终哈希)       |
  |                               |
  |<------ transfer_result -------|
  |     (成功/失败、文件路径)      |
  |                               |
```

### 下载流程

```
客户端                          设备端
  |                               |
  |--- file_download_request ---->|
  |     (文件 ID 或路径)            |
  |                               |
  |<--- file_transfer_start ------|
  |     (文件信息)                |
  |                               |
  |<--- file_transfer_chunk -----|
  |     (分块 0)                   |
  |                               |
  |<--- file_transfer_chunk -----|
  |     (分块 1)                   |
  |                               |
  |          ...                  |
  |                               |
  |<--- file_transfer_complete -->|
  |     (完成信号)                |
  |                               |
  重组文件并验证哈希
```

### 2. 压缩传输

```python
import gzip

def compress_chunk(data):
    """压缩分块数据"""
    if len(data) > 1024:  # 大于 1KB 才压缩
        return gzip.compress(data)
    return data
```

### 3. 流量控制

```python
# 限制传输速度
async def upload_with_rate_limit(data, limit_bps):
    start = time.time()
    await send(data)
    elapsed = time.time() - start
    expected = len(data) / limit_bps
    
    if elapsed < expected:
        await asyncio.sleep(expected - elapsed)
```

## 🔍 故障排查

### 常见问题

#### 1. 传输中断

**原因**: 网络不稳定、超时

**解决**:
```bash
# 使用断点续传
python3 tools/file_transfer_cli.py upload file.txt -d device-001 --resume
```

#### 2. 哈希不匹配

**原因**: 传输过程中数据损坏

**解决**:
- 检查网络连接
- 减小分块大小
- 增加重试次数

#### 3. 速度慢

**原因**: 网络带宽限制、分块太小

**解决**:
```python
# 增大分块
CHUNK_SIZE = 4 * 1024 * 1024  # 4MB

# 启用压缩
ENABLE_COMPRESSION = True

# 增加并发
CONCURRENCY = 8
```
