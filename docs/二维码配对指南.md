# AgentLinker 二维码配对 & 局域网发现使用指南

## 🎯 新功能

### 1. 二维码配对
扫描设备上的二维码即可完成配对，无需手动输入设备 ID 和配对密钥。

### 2. 局域网自动发现
自动发现同一局域网内的设备，快速连接本地主机。

---

## 📱 二维码配对流程

### 方式一：设备端生成二维码

**在被控端主机上：**

```bash
# 查看配对密钥并生成二维码
agentlinker show-qr

# 或者手动生成
agentlinker pairing qr-gen \
  --device-id my-server \
  --pairing-key XK9M2P7Q \
  --device-name "我的服务器"
```

**在主控端：**

```bash
# 启动二维码配对
agentlinker --mode controller

# 进入交互模式后输入：
[controller]> qr-pair

# 然后扫描设备上的二维码
```

### 方式二：从日志中获取二维码

设备启动时会在日志中显示配对密钥和二维码：

```bash
# 查看设备日志
sudo journalctl -u agentlinker -f

# 输出示例：
==================================================
🔐 配对密钥已生成！
   设备 ID: my-server-01
   配对密钥：XK9M2P7Q
   
📱 二维码内容:
{"type":"agentlinker_pair","device_id":"my-server-01","pairing_key":"XK9M2P7Q"}
==================================================
```

复制二维码内容，在主控端使用：

```bash
agentlinker pairing qr-scan '{"type":"agentlinker_pair","device_id":"my-server-01","pairing_key":"XK9M2P7Q"}'
```

## 📡 局域网发现流程

### 自动发现设备

**在主控端：**

```bash
# 启动控制器
agentlinker --mode controller

# 进入交互模式后输入：
[controller]> discover

# 或指定超时时间（秒）
[controller]> discover 15
```

系统会自动搜索局域网内的设备并列出：

```
📡 正在搜索局域网设备...

📱 发现设备：My-Server
   IP: 192.168.1.100:8080
   平台：Linux 5.15.0
   配对密钥：XK9M2P7Q

📱 发现设备：MacBook-Pro
   IP: 192.168.1.101:8080
   平台：macOS 14.0
   配对密钥：P3K8M2X9

搜索完成，共发现 2 个设备

选择设备进行配对:
  1. 🟢 My-Server @ 192.168.1.100
  2. 🟢 MacBook-Pro @ 192.168.1.101

选择设备编号 (或 q 退出): 1

正在配对 My-Server...
✅ 配对成功!
```

### 命令行发现

```bash
# 发现设备
agentlinker pairing discover --timeout 10

# 交互式发现（可选择设备配对）
agentlinker pairing discover -i
```

## 🔧 高级用法

### 生成二维码图片

```bash
# 保存为 PNG 图片
agentlinker pairing qr-gen \
  --device-id my-server \
  --pairing-key XK9M2P7Q \
  --device-name "我的服务器" \
  -o pairing_qr.png

# 然后用手机扫描图片
```

### 广播设备信息

**在被控端：**

```bash
# 广播设备信息（包含配对密钥）
agentlinker pairing broadcast \
  --device-id my-server \
  --device-name "我的服务器" \
  --pairing-key XK9M2P7Q
```

这会让设备在局域网内广播自己，方便主控端发现。

### 自定义发现参数

```python
# Python 代码中使用
from client.utils.discovery import LANDiscovery

discovery = LANDiscovery(
    device_id="my-device",
    device_name="My Device"
)

def on_found(device):
    print(f"发现设备：{device.device_name} @ {device.ip_address}")

discovery.on_device_found = on_found
discovery.start_listener(timeout=10.0)
```

## 🌐 网络配置

### 防火墙设置

确保以下端口开放：

- **UDP 53535** - 局域网发现（广播/组播）
- **TCP 8080** - WebSocket 连接（可自定义）

**Linux (ufw):**
```bash
sudo ufw allow 53535/udp
sudo ufw allow 8080/tcp
```

**macOS:**
```bash
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /path/to/python
```

### 跨网段发现

默认情况下，广播只能在同一网段内工作。跨网段需要：

1. 配置路由器转发 UDP 广播
2. 或使用组播（已支持）
3. 或手动添加设备

## 📋 完整示例

### 场景：控制家里的多台电脑

**1. 在被控端安装并启动：**

```bash
# 电脑 A (Linux)
curl -fsSL https://your-server.com/install.sh | sudo bash

```

**3. 批量控制：**

```bash

```

## 🔐 安全提示

1. **配对密钥** - 只在首次配对时需要，配对后持久化保存
2. **二维码时效** - 建议每次重启后生成新二维码
3. **局域网信任** - 确保在可信网络中使用发现功能
4. **生产环境** - 建议手动配置设备，关闭自动发现

## 🛠️ 故障排除

### 设备无法被发现

1. 检查防火墙是否开放 UDP 53535
2. 确认设备在同一网段
3. 检查路由器是否隔离了客户端

### 二维码无法扫描

1. 确保二维码内容完整
2. 检查二维码版本（v1）
3. 尝试手动输入配对密钥

### 配对失败

1. 确认配对密钥正确（8 位，区分大小写）
2. 检查设备是否在线
3. 查看服务端日志

## 📚 API 参考

### Python API

```python
from client.utils.discovery import LANDiscovery, QRCodePairing

# 局域网发现
discovery = LANDiscovery()
discovery.on_device_found = lambda d: print(f"发现：{d.device_name}")
discovery.start_listener(timeout=10)

# 二维码生成
qr = QRCodePairing()
qr_content = qr.generate_pairing_qr(
    device_id="my-device",
    pairing_key="XK9M2P7Q"
)
qr.generate_qr_image(qr_content, "qr.png")
```
